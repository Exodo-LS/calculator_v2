Useful Glossary Terms

Here are some useful terms to make understand this a lot easier

  • Object Oriented Programming - A programming approach that uses the implementation of objects to solve problems
  • Class - Blueprint of the object
  • Object - An instantiation of a class
  • Instantiation - The process of taking a class definition and creating an object that you can use in a program
  • Method - Functions defined in a class
  • Static - The member is on a class level rather on the instance level (can be variable or function)
  • Class Method - A method that is bound to the class and not the object of the class
  • Constructor - A special type of member function of a class which initializes objects of a class
  • Factory - An object for creating other objects
  • Namespace - A declarative region that provides a scope to the identifiers inside it
  • Fixture - A function that will run each time you pass it to a test
  • Type Hint - A formal solution to statically indicate the type of value within your code
  • Type Cast - When you assign a value of one primitive data type to another type
  • Unit Test - Validates that your functions are fit for use
  • Facade - An object that provides a simplified interface to a larger body of code, such as a class library
  • Design Patterns Used - A repeatable solution to a software engineering problem
  • Property - Are characteristics or attributes of an object
  • Inheritance - Creating a new class for using details of an existing class without modifying it
  • Encapsulation - Prevents data from direct modification
  • Abstraction - Hiding the unnecessary details from the users
  • Polymorphism - The ability to use a common interface for multiple forms
